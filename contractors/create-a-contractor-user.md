# Créer un utilisateur sous-traitantPage 1

Nomadia Delivery introduit un nouveau type d’utilisateur appelé Prestataire. Ce type d’utilisateur peut importer des missions, gérer des listes d’adresses et suivre l’avancement des missions, offrant ainsi aux prestataires un meilleur contrôle sur leur travail et améliorant l’efficacité globale.

Pour créer un nouveau type d’utilisateur Prestataire, suivez ces étapes.

1. Ouvrez l’application Nomadia Delivery et allez dans Configuration > Gérer les utilisateurs

<figure><img src="../.gitbook/assets/image (415).png" alt=""><figcaption></figcaption></figure>

2. Cliquez sur le bouton Action et sélectionnez Ajouter

<figure><img src="../.gitbook/assets/image (416).png" alt=""><figcaption></figcaption></figure>

3. Cliquez sur OK pour créer un nouvel utilisateur. Vous pouvez également créer un utilisateur à partir d’un utilisateur existant, ce qui accélère le processus en copiant tous les paramètres de l’utilisateur existant vers le nouveau profil.

<figure><img src="../.gitbook/assets/image (417).png" alt=""><figcaption></figcaption></figure>

4. Dans l’onglet Généralités, saisissez le login et le nom de famille de l’utilisateur, activez le statut de l’utilisateur, puis passez à l’onglet Accès

<figure><img src="../.gitbook/assets/image (418).png" alt=""><figcaption></figcaption></figure>

5. Activez l’accès web pour l’utilisateur et sélectionnez le site du prestataire dans le menu déroulant

<figure><img src="../.gitbook/assets/image (419).png" alt=""><figcaption></figcaption></figure>

6. Sélectionnez le prestataire approprié (créé dans la section précédente de ce document) à associer à l’utilisateur, puis passez à l’onglet Rôles et droits

**Remarque**: Si aucun prestataire n’apparaît dans la liste, assurez-vous qu’un a bien été créé dans le module Prestataires.<br>

<figure><img src="../.gitbook/assets/image (420).png" alt=""><figcaption></figcaption></figure>

7. Attribuez les droits nécessaires au nouvel utilisateur Prestataire, puis passez à l’onglet Agences

<figure><img src="../.gitbook/assets/image (421).png" alt=""><figcaption></figcaption></figure>

<br>

8. Cliquez sur Enregistrer

<figure><img src="../.gitbook/assets/image (422).png" alt=""><figcaption></figcaption></figure>

9. Cliquez sur le lien pour vérifier le compte
10. Le Prestataire recevra un e-mail contenant les instructions pour définir un mot de passe et activer le compte

<figure><img src="../.gitbook/assets/image (423).png" alt=""><figcaption></figcaption></figure>

Saisissez le code de vérification reçu par e-mail et cliquez sur Vérifier le code.

<figure><img src="../.gitbook/assets/image (424).png" alt=""><figcaption></figcaption></figure>

Cliquez sur Continuer.

<figure><img src="../.gitbook/assets/image (425).png" alt=""><figcaption></figcaption></figure>

Saisissez et confirmez le nouveau mot de passe, puis cliquez sur Continuer.

<figure><img src="../.gitbook/assets/image (426).png" alt=""><figcaption></figcaption></figure>

Relisez et acceptez le Contrat de licence utilisateur final (EULA), puis cliquez sur Continuer.

<figure><img src="../.gitbook/assets/image (427).png" alt=""><figcaption></figcaption></figure>

Le nouvel utilisateur Prestataire peut maintenant accéder à l’application et commencer à effectuer des tâches

<figure><img src="../.gitbook/assets/image (428).png" alt=""><figcaption></figcaption></figure>
