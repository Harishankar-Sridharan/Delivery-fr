# Créneaux de réservation

Le portail de réservation I-Frame est conçu pour offrir aux transporteurs une solution de réservation flexible et fluide, pouvant être directement intégrée aux sites web des sous-traitants. Son objectif est de renforcer l’engagement client en permettant la réservation optimisée de créneaux de livraison et en offrant une expérience utilisateur sans friction.

Pour configurer Nomadia Delivery pour le portail de réservation de rendez-vous, les transporteurs doivent suivre les étapes suivantes :

1. Ouvrez l'application Nomadia Delivery et accédez à l'onglet **Configuration** .
2. Dans le menu déroulant, sélectionnez **Configurer les pages d’expérience client**.

<figure><img src="../../.gitbook/assets/image (71).png" alt=""><figcaption></figcaption></figure>

3. Choisissez le **Ajouter un élément de menu de livraison programmée**.
4. Ouvrez le **Onglet du portail de réservation**.
5. Activer le bouton **Afficher le portail de réservation.**
6. Configurez les champs que vous souhaitez afficher au client final afin de recueillir les informations nécessaires à la bonne livraison. Par défaut, Nom et Adresse sont obligatoires, car ils sont essentiels pour localiser le client.
7. Cliquez sur **Ajouter un champ** pour inclure des champs supplémentaires.

<figure><img src="../../.gitbook/assets/image (72).png" alt=""><figcaption></figcaption></figure>

8. Spécifiez le **nom du champ** et sélectionnez la **type de champ** **(Texte, Case à cocher, Texte multiligne, Liste de valeurs, Nombre ou date).**

Exemple : Nom du champ – Code d’accès (le code dont le livreur a besoin pour accéder aux locaux du client).

9. Choisissez le mode du champ, comme Saisie obligatoire, Masqué, Lecture seule ou Lecture/écriture, afin de définir le niveau de contrôle du client lors du remplissage du formulaire.

<figure><img src="../../.gitbook/assets/image (73).png" alt=""><figcaption></figcaption></figure>

10. Cliquez sur **Ajouter** pour inclure le champ personnalisé dans le formulaire.

<figure><img src="../../.gitbook/assets/image (74).png" alt=""><figcaption></figcaption></figure>

Le champ apparaîtra désormais dans la liste des données à collecter auprès du client.

<figure><img src="../../.gitbook/assets/image (75).png" alt=""><figcaption></figcaption></figure>

Pour intégrer le portail de réservation au site web du sous-traitant, les transporteurs doivent suivre ces étapes : Copier l’exemple de code depuis le portail de réservation.

<figure><img src="../../.gitbook/assets/image (76).png" alt=""><figcaption></figcaption></figure>

Le partager avec le sous-traitant pour intégration.

L’URL dans l’exemple de code redirige automatiquement vers le portail de réservation et peut être intégrée dans une I-frame sur le site web du sous-traitant.

<figure><img src="../../.gitbook/assets/image (77).png" alt=""><figcaption></figcaption></figure>

L’I-frame permet aux clients de planifier directement les livraisons.

Le client peut cliquer sur Programmer pour afficher en temps réel les créneaux horaires optimisés, puis sélectionner son créneau préféré en cliquant sur Choisir.

<figure><img src="../../.gitbook/assets/image (78).png" alt=""><figcaption></figcaption></figure>

Le portail de réservation est directement lié aux plans de tournée des chauffeurs dans l’application Nomadia Delivery. Une fois la réservation confirmée :

* Une mission est automatiquement créée.
* La mission est planifiée de manière optimale et ajoutée à la tournée en temps réel du chauffeur.

<figure><img src="../../.gitbook/assets/image (79).png" alt=""><figcaption></figcaption></figure>

Le portail de réservation I-Frame aide les transporteurs à renforcer leur présence numérique et l’engagement client

* **en proposant**
* **Une intégration fluide**
* **Des formulaires de réservation personnalisables**
* **Une administration facile**
* **Des créneaux de livraison optimisés**
* **Des notifications de confirmation de créneau**
