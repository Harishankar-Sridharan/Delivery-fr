# Gérer la configuration des sous-statuts

#### GESTION DES TYPES DE CONFIGURATION DES SOUS-STATUTS

Nomadia Delivery propose trois types de modèles de sous-statuts qui permettent aux utilisateurs de personnaliser le flux de livraison selon le type ou la catégorie de client. Ces modèles sont : Basique, Intermédiaire et Avancé.

Types de configuration basiques : conçus pour les clients qui n’ont besoin que d’une seule étape de vérification,

comme la prise d’une photo ou la collecte d’une signature. Cela simplifie le processus de livraison en incluant

uniquement les contrôles essentiels.

Types de configuration intermédiaires : adaptés aux clients qui ont besoin de plusieurs étapes de vérification, mais qui n’exigent pas toute la gamme d’options disponibles dans le modèle avancé.

Types de configuration avancés : destinés aux clients qui exigent une vérification complète, comme la prise de photos à chaque étape de la livraison. Ils offrent une flexibilité maximale pour configurer des flux de travail détaillés.

Remarque : les options de types de configuration sont fournies uniquement pour aider à différencier les flux de livraison en fonction des catégories de clients. Cependant, la complexité du flux de travail relève entièrement de la discrétion de l’utilisateur. Par exemple, un modèle Basique peut tout de même être configuré pour prendre en charge un processus de livraison complexe si nécessaire.

Pour configurer et gérer les types de configuration des sous-statuts dans Nomadia Delivery, suivez ces étapes

1. Ouvrez l'application Nomadia Delivery et accédez à l'onglet **Configuration** module.

<figure><img src="../.gitbook/assets/image (646).png" alt=""><figcaption></figcaption></figure>

2. Cliquez sur le **Actions** bouton, puis choisissez **Ajouter** pour créer un nouveau sous-statut.

<figure><img src="../.gitbook/assets/image (647).png" alt=""><figcaption></figcaption></figure>

3. Attribuez le sous-statut créé à un ou plusieurs modèles.

<figure><img src="../.gitbook/assets/image (648).png" alt=""><figcaption></figcaption></figure>

4. Définissez les actions requises pour chaque sous-statut, comme rendre la signature obligatoire, exiger une photo, etc.

<figure><img src="../.gitbook/assets/image (649).png" alt=""><figcaption></figcaption></figure>

5. Répétez le processus pour créer des sous-statuts supplémentaires et les lier aux modèles souhaités.

<figure><img src="../.gitbook/assets/image (650).png" alt=""><figcaption></figcaption></figure>

6. Une fois tous les modèles configurés, accédez à la **Missions** .
7. Cliquez sur le **Mission** bouton Éditeur pour associer un modèle de sous-statut à une mission.

<figure><img src="../.gitbook/assets/image (651).png" alt=""><figcaption></figcaption></figure>

8. Sélectionnez le **modèle de configuration de statut** pour la mission choisie

<figure><img src="../.gitbook/assets/image (652).png" alt=""><figcaption></figcaption></figure>
